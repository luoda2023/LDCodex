# LUODA 路由中转平台 · 使用手册

## 平台概述

LUODA 是一个 AI 模型路由中转代理平台，提供智能模型调度、自动轮循切换、异常隔离等功能。平台基于 Node.js 22 + SQLite 构建，分为转发代理和管理平台两部分。

### 核心架构

```
管理平台 (port 40002) ←→ SQLite 数据库 ←→ 中间人 (Middleman)
                                          ↓
转发代理 (port 40000) ←→ JSON 配置文件 ←──┘
```

- **管理平台**: 提供 Web UI (models.html / index.html) 进行模型管理
- **转发代理**: 处理上游 API 请求，实现路由转发
- **中间人**: 实时同步 SQLite → JSON，确保配置一致性
- **数据库**: 所有配置存储于 `data/admin.db`（SQLite）

## 核心功能

### 1. 模型管理 (models.html)

#### 模型列表
- 显示所有已注册的 AI 模型（内置 + 自定义）
- 支持按名称搜索、按状态/类型筛选
- 每页显示 12 个模型，支持翻页

#### 新增模型
1. 切换到「添加模型」Tab
2. 填写：模型名称、API 地址、API 密钥、模型 ID
3. 点击「保存模型」→ 即时写入数据库 → 即时生效

#### 编辑模型
1. 在模型列表点击「编辑」
2. 修改任意字段（名称、地址、密钥、模型 ID）
3. 点击「确认保存」→ 即时更新数据库 → 同步到转发代理

#### 删除模型
1. 点击「删除」→ 确认
2. 自动清除 CODEX/HERMES 锁定引用
3. 从数据库移除 + 从轮循队列移除

### 2. 自动轮循队列

#### 查看顺序
- 页面顶部「自动轮循队列」区域显示所有在线模型的轮循顺序
- 彩色圆点标识模型（1, 2, 3...）

#### 拖动排序
- 拖拽模型标签调整顺序 → 即时保存到数据库
- 刷新页面后顺序保持不变

#### 双击设置当前模型
- **双击**: 设置 CODEX 当前模型（亮红色标记）
- **10秒内双击另一个模型**: 设置 HERMES 当前模型（黄色标记）
- CODEX 和 HERMES **禁止使用同一个模型**（铁律）
- 设置即时写入数据库，重启不丢失

#### 异常模型
- 标记异常后：从正常列表消失 → 从轮循队列移除 → 仅出现在异常 Tab
- 取消异常后：恢复到正常列表 → 追加到轮循队列末尾

### 3. 自动切换逻辑

#### 切换条件（仅2种）
1. **自动开关打开时（ON）**：倒计时归零 → CODEX + HERMES **同时**各前进1位
2. **自动开关关闭时（OFF）**：倒计时冻结，仅额度用完触发切换

#### 额度用完切换（不受开关影响）
| 触发条件 | 等待时间 | 行为 |
|----------|---------|------|
| **额度用完**（quota/402/insufficient） | **立即** | 标记异常 + 清除锁定 + 前进1位 |
| **限速**（rate limit） | **3分钟**（180秒） | 3分钟后仍限速 → 切换 |
| **超时/4xx/5xx 其他错误** | **3分钟**（180秒） | **3分钟后仍错误 → 切换** |

#### 切换规则
- 每次只能切换 **1个模型**（不可跳跃，最多试下一位）
- 切换方向：从左往右
- CODEX 和 HERMES 切换时自动检测冲突，**不会落在同一个模型上**
- 额度用完时返回给 CODEX 的提示：
  - 额度用完 → ⚠️ 当前模型额度已用完！正在切换，请稍等...
  - 限速中 → ⏳ 请稍候... 正在限速中！！
  - 其他错误 → 🔄 请稍候... 正在切换中！！

#### 服务端倒计时守护
- 服务器启动后自动运行倒计时检测（每10秒），**不依赖前端页面**
- 倒计时时间存储在数据库 `config_kv._countdown_start`，重启不丢失
- 所有页面（index.html / models.html）从服务端读取剩余时间，仅做展示

#### 单模型锁定保护
- 倒计时切换：**自动清除锁定**（之前规则已取消）
- 额度用完切换：同样清除锁定
- CODEX 和 HERMES **禁止使用同一个模型**（按 slug 名称 + modelId 双保险）

### 4. 控制台 (index.html)

#### 统计卡片
- **全部模型**: 历史 Token 总消耗
- **neizhiAPI / CodexAPI / HermesAPI**: 按通道统计
- 当前模型实时显示（与数据库同步）

#### Token 曲线图
- 显示近1小时 / 近1天 / 近1个月的 Token 消耗趋势
- 智能默认：有近1小时数据→显示小时；否则近1天；否则近1个月

#### 模型使用排行
- 按 Token 调用量从高到低排列
- 最多显示 8 个模型（从 SQLite 数据库读取）
- 带进度条和数值显示

### 5. 异常模型处理

#### 自动异常
- 连续4次额度用完 → 自动标记异常
- 从轮循队列移除 → 从正常列表删除 → 进入异常列表
- 同步写入数据库

#### 手动异常
- 在模型列表点击「标记异常」
- 行为与自动异常一致

#### 恢复异常
- 在异常 Tab 点击「取消异常」
- 恢复到正常列表 + 追加到轮循队列末尾

## 启动与停止

### 启动主服务
```bash
cd /j/luoda-bridge
PROXY_PORT=40000 CONFIG_PORT=40001 ADMIN_PORT=40002 nohup node index.mjs > data/luoda.log 2>&1 &
```

### 启动独立管理服务
```bash
cd /j/luoda-bridge
nohup node manage.mjs > data/manage.log 2>&1 &
```

### 停止服务
```bash
# 查找 PID
ps aux | grep node
# 或使用 Windows 任务管理器
```

## 数据库说明

### 位置
- `data/admin.db` — SQLite 主数据库

### 表结构
| 表名 | 说明 |
|------|------|
| `models` | 模型列表（名称、slug、API地址、密钥、模型ID、顺序索引） |
| `config_kv` | 配置键值对（fallback_sequence、single_model_codex、abnormal_models 等） |
| `db_version` | 版本跟踪（中间人同步检测） |
| `provider_tokens` | Provider Token 消耗统计（排行数据源） |

### 配置键
| 键 | 说明 |
|----|------|
| `fallback_sequence` | 轮循队列顺序（分号分隔的 slug） |
| `single_model_codex` | CODEX 锁定模型 |
| `single_model_hermes` | HERMES 锁定模型 |
| `abnormal_models` | 异常模型列表 |
| `cond_switch_enabled` | 自动切换开关 |
| `fallback_interval_minutes` | 切换间隔（分钟） |
| `_fallbackState` | fallback 运行时状态 |

## 常见问题

### Q: 编辑模型后不生效？
A: 检查浏览器缓存（Ctrl+F5）。PUT 请求会写入 DB + 同步 middleman + 重注册 provider，即时生效。

### Q: CODEX 显示 Reconnecting？
A: 代理在额度用完时会返回「请稍等」响应，5分钟无响应会自动切换模型。

### Q: 模型异常后如何恢复？
A: 在「异常模型」Tab 中点击「取消异常」，模型会自动回到正常列表和轮循队列末尾。

### Q: 端口被占用？
A: 修改 `.env` 文件中的 `PROXY_PORT` / `CONFIG_PORT` / `ADMIN_PORT` 后重启服务。

### Q: CODEX 无法连接？
A: 确认 CODEX 配置的 base_url 为 `http://127.0.0.1:40000`，API Key 为 `.env` 中的 `PROXY_AUTH_KEY`。
